# Bootcamp Drills

## 1. Why I Joined This Bootcamp
I joined this bootcamp because I want to become a professional web developer who can confidently build full-stack applications. I want to move from just learning theory to actually creating real projects, improving my problem-solving skills, and learning how to work like an engineer in a structured, daily practice environment.

---

## 2. Skills I Want to Gain Confidence In
1. **Full-stack web development:** Building and deploying applications with TypeScript and Node.js. I’m not very familiar with TypeScript and Next.js, so I hope to learn a lot about them.  
2. **Version control:** Using Git effectively and collaborating on code with others.  
3. **Debugging:** Troubleshooting complex problems independently, without relying on LLMs, which I tend to use a lot.

---

## 3. What a Good Day of Learning Looks Like
A good day of learning includes: setting a clear goal in the morning and completing focused coding drills.

---

## 4. Measurable Goals

**Technical:**  
Build and deploy a functional web application with authentication by the end of week 5, entirely on my own and without using LLMs. I also want to learn new tools and frameworks that will help me in the future.

**Professional:**  
Commit to Git daily with proper commit messages and a structured workflow, and upload daily updates in Slack by 12 PM or earlier.

**Personal:**  
Spend at least 12 hours each day debugging and documenting what I learned, tracking progress in a journal. As Pooja said, even 12 hours is not enough for this bootcamp.

---

## 5. Habits from School I Won’t Rely On
1. Not paying attention to deadlines — I need to be very careful about them.  
2. Copying solutions from others instead of figuring things out myself.  
3. Just aiming to pass — I will try my best to top this bootcamp.
